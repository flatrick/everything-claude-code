# Merge continuous-learning-* into ai-learning

- Status: `not-started`
- Created: `2026-03-13`
- Last updated: `2026-03-13`
- Owner: `mdt`
- Execution mode: `agent`
- Related docs: `skills/continuous-learning-automatic/SKILL.md`, `skills/continuous-learning-manual/SKILL.md`, `codex-template/skills/continuous-learning-manual/SKILL.md`, `BACKLOG.md`

## Goal

- [ ] Replace `continuous-learning-automatic` and `continuous-learning-manual` with a single `ai-learning` skill that adapts its observation and analysis path to the environment: hooks-passive for Claude/Cursor, manual/observer-optional for Codex and hook-free tools.

## Non-goals

- [ ] Do not merge `/learn` and `/learn-eval` commands — they serve distinct use-cases (fast capture vs quality-gated extraction with rubric).
- [ ] Do not enable the observer by default in the Codex template — the observer is always optional in hook-free environments.
- [ ] Do not change the instinct storage schema, confidence model, or project-detection logic.
- [ ] Do not remove the `mdt learning` CLI surface (capture, analyze, status, retrospective, observer).
- [ ] Do not change the instinct management commands (instinct-status, evolve, instinct-export, instinct-import, promote, projects).

## Open Questions / Assumptions

- [x] `/learn` and `/learn-eval` stay separate: confirmed by reviewing both commands — they are not equivalent.
- [x] `codex-template` keeps `agents/openai.yaml` — it is Codex-specific and has no counterpart in the shared skills.
- [x] Observer enabled by default for Claude/Cursor (`config.json` `observer.enabled: true`), disabled by default for Codex (`codex-template` `config.json` `observer.enabled: false`).
- [x] No `hooks/` directory in the codex-template version of `ai-learning` — Codex does not use MDT hooks.
- [x] `detect-project.js` appears in both skills; the copies are functionally identical — keep one copy in `ai-learning/scripts/`.

## Decisions

- [x] New skill name: `ai-learning` (replaces both `continuous-learning-automatic` and `continuous-learning-manual`).
- [x] Single `skill.meta.json`: hooks mode `optional`, tools `["claude", "cursor"]` — installs everywhere, hooks used when available.
- [x] Running the `/learn` command or `mdt learning` in a hook-enabled environment triggers the same analysis pipeline as the background observer — there is no separate "automatic" code path; hooks are only the observation capture source.
- [x] The external observer fills the hook role in hook-free environments when opted in — no special code path needed, same observer binary.
- [x] codex-template gets its own `skills/ai-learning/` mirroring the main skill minus `hooks/` plus `agents/openai.yaml`.
